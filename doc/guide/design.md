# ゲームコンシェルジュ UI/UX設計ドキュメント

## 1. 設計概要

### 1.1 コンセプト
「ゲーム体験をもっと深く、豊かに味わうためのプラットフォーム」

### 1.2 価値提案
ゲーム体験の記録・可視化により、ゲームをより楽しくする

### 1.3 ターゲットユーザー
Nintendo Switchを中心とするライトゲーマー
- プレイ時間：1-3時間/日
- プレイタイミング：夕方・夜・休日
- 特徴：限られた時間で効率的にゲームを楽しみたい

### 1.4 核心インサイト
1. **移行期の迷い**：「ゲームを終えたけど、次に何をしようかわからない」
2. **進行状況の忘却**：「中途半端なゲームがあるけど、どこまで進んでたかわからない」
3. **選択の困難**：「気分転換にゲームしたいけど、どれもピンとこない」

### 1.5 設計理念
- **エンタメ領域の特性**：課題解決型ではなく体験向上型
- **ショッピング体験**：選ぶこと自体が楽しい体験の創出
- **記憶の補完**：「候補に漏れがある」問題の解決
- **わかりやすさ最優先**：Switch UIの良さを参考にしつつ、スマホ特性を考慮

## 2. ユーザー行動分析

### 2.1 現在のゲーム選択プロセス
1. 頭の中の候補を検討
2. Switch本体のeShopをチェック
3. 所持パッケージ版ソフトを確認
4. （漏れがあることに後から気づく）

### 2.2 課題
- **記憶依存の限界**：「候補に漏れがある」「あとから思い出す」
- **情報の分散**：複数の場所に情報が散在
- **状況の把握困難**：100-200本のコレクション管理が困難

### 2.3 理想的な体験
- 自分に合ったゲームが即座にリストアップ・レコメンド
- 「次はこれをやろう」とすぐに決められる
- ゲーム選択プロセス自体が楽しい

## 3. OOUI設計プロセス

### 3.1 ユーザータスク（動詞）の抽出

#### 3.1.1 メインシナリオ（優先度：高）
1. **所持ゲーム管理**：所持・プレイ中・気になるゲームのリスト管理
2. **気分に合うゲーム探し**：今の気分・状況に適したゲームの発見
3. **ゲーム別メモ**：進行状況・やりたいことリストの記録

#### 3.1.2 サブシナリオ（優先度：中）
- 新発売ゲーム情報の取得
- セール情報の確認
- 評判・レビューの調査
- 攻略ヒント・RTAの視聴

#### 3.1.3 コミュニティ機能（優先度：低）
- 感想のやりとり
- 配信者へのリクエスト
- インディーゲーム応援

### 3.2 オブジェクト（名詞）の特定

#### 3.2.1 メインオブジェクト
1. **ゲーム**：アプリの中心的存在
2. **ユーザー状況**：各ゲームに対するユーザーの関係性・状態
3. **ニュース**：ゲーム関連の最新情報

#### 3.2.2 サブオブジェクト
- メモ・記録
- セール情報
- 評判・レビュー
- 配信動画

### 3.3 状態設計

#### 3.3.1 所持状況
- **所持済み**：ユーザーが所有しているゲーム
- **気になるリスト**：未所持だが興味があるゲーム

#### 3.3.2 プレイ状況
- **プレイ中**：現在進行中のゲーム
- **中断中**：一時的に中断しているゲーム
- **クリア済み**：ストーリーを完了したゲーム
- **やり込み中**：クリア後も継続してプレイしているゲーム

## 4. インタラクション設計

### 4.1 基本操作パターン
- **タップ**：詳細画面への遷移
- **ボタン操作**：状況の変更、お気に入り追加等の明示的な操作

### 4.2 「名詞→動詞」の操作原理
1. ゲームを選ぶ → 詳細を見る / メモを書く / 状況を変更する
2. 状況を選ぶ → 一覧を絞り込む / 並び替える
3. メモを選ぶ → 編集する / 削除する

### 4.3 状態変更の流れ
```
未所持 → 気になるリスト → 所持済み
      ↓
プレイ中 ⇄ 中断中 → クリア済み → やり込み中
```

## 5. 画面設計

### 5.1 一覧画面（メイン画面）

#### 5.1.1 表示要素
- **ゲーム画像**：パッケージ・アイコン画像
- **タイトル**：ゲーム名
- **進行状況**：現在の状態（プレイ中/中断中/クリア済み等）
- **状態インジケーター**：色・アイコンによる視覚的区別

#### 5.1.2 機能
- **並び替え**：追加日、タイトル順
- **絞り込み**：プレイ状況、所持状況による フィルタリング
- **検索**：タイトル・ジャンルによる検索
- **お気に入り追加**：気になるゲームをリストに追加するボタン

### 5.2 詳細画面

#### 5.2.1 所持済みゲーム
- **進行状況セクション**：現在の進行度、プレイ時間
- **メモセクション**：ユーザーが記録したメモ・TODO
- **状況変更ボタン**：プレイ中/中断中/クリア済み の切り替え
- **基本情報**：ゲーム概要、開発者情報

#### 5.2.2 未所持ゲーム
- **ゲーム情報**：概要、動画、スクリーンショット
- **購入情報**：価格、セール情報、購入リンク
- **レビュー**：評価、ユーザーレビュー
- **類似ゲーム**：関連するゲームの提案

### 5.3 視覚的状態表現

#### 5.3.1 色による区別
- **プレイ中**：青系（アクティブ）
- **中断中**：オレンジ系（注意）
- **クリア済み**：緑系（完了）
- **気になるリスト**：グレー系（未所持）

#### 5.3.2 アイコンによる区別
- **プレイ中**：再生アイコン（▶️）
- **中断中**：一時停止アイコン（⏸️）
- **クリア済み**：完了アイコン（要検討）
- **やり込み中**：継続アイコン（要検討）

## 6. データモデル設計

### 6.1 ゲームテーブル
```
Game {
  id: string
  title: { [locale: string]: string }  // 多言語対応
  imageUrl: { [locale: string]: string }
  description: { [locale: string]: string }
  genre: string[]
  developerId: string  // 開発者テーブルとのリレーション
  releaseDate: Date
  platform: ('nintendo_switch' | 'steam' | 'ps5' | 'xbox')[]
  externalIds: {
    igdb?: string
    steam?: string
    nintendo?: string
  }
}
```

### 6.1.1 開発者テーブル
```
Developer {
  id: string
  name: string
  description?: string
  website?: string
  logoUrl?: string
}
```

### 6.2 ユーザーゲーム関係テーブル
```
UserGame {
  userId: string
  gameId: string
  status: 'playing' | 'paused' | 'completed' | 'grinding' | 'wishlist'
  ownStatus: 'owned' | 'wishlist'
  // progress: 一旦削除（取得方法未定）
  // playTime: 一旦削除（取得方法未定）
  // lastPlayedAt: 一旦削除（取得方法未定）
  // notes: 別テーブルで管理
  tags: string[]
  createdAt: Date
  updatedAt: Date
}
```

### 6.3 メモテーブル
```
GameNote {
  id: string
  userId: string
  gameId: string
  title: string
  content: string
  type: 'progress' | 'todo' | 'tip' | 'review'
  createdAt: Date
  updatedAt: Date
}
```

## 7. MVP機能仕様

### 7.1 Phase 1（最優先）
1. **ゲーム検索・追加機能**
   - 外部APIからのゲーム情報取得
   - 自分のコレクションへの追加
   - 基本情報の表示

2. **ゲーム一覧・管理機能**
   - 所持ゲームの一覧表示
   - 状態（プレイ中/中断中/クリア済み）の管理
   - 並び替え・絞り込み

3. **基本メモ機能**
   - ゲームごとのシンプルなメモ機能
   - 進行状況の記録

### 7.2 Phase 2（中期）
1. **気分マッチング機能**
   - 現在の気分に合うゲームの提案
   - タグベースのレコメンド

2. **セール情報連携**
   - 気になるゲームのセール通知
   - 価格履歴の表示

3. **詳細メモ機能**
   - TODO管理
   - 攻略メモ
   - 感想記録

### 7.3 Phase 3（長期）
1. **AIテンプレート機能**
   - ゲーム特化のメモテンプレート自動生成
   - 写真解析による記録支援

2. **コミュニティ機能**
   - 感想の共有
   - レコメンドの交換

## 8. 技術的実現性

### 8.1 外部API連携
- **IGDB API**：ゲーム情報の取得
- **Steam API**：PC版ゲーム情報（該当する場合）
- **価格.com API**：価格情報（可能であれば）

### 8.2 Switch連携の代替案
- **手動入力**：ユーザーによる状況更新
- **インポート機能**：写真からのゲームリスト読み取り（将来的）
- **QRコード読み取り**：パッケージからの情報取得（将来的）

### 8.3 データ同期
- **クラウド同期**：複数デバイス間でのデータ共有
- **ローカルファースト**：オフラインでも基本機能が使用可能
- **バックアップ機能**：データの安全性確保

## 9. 成功指標

### 9.1 定量的指標
- **DAU（Daily Active Users）**：日次アクティブユーザー数
- **ゲーム追加数**：ユーザーがコレクションに追加したゲーム数
- **メモ作成数**：作成されたメモの総数
- **状態更新頻度**：ユーザーがゲーム状態を更新する頻度

### 9.2 定性的指標
- **習慣化の実現**：「ゲームを検討・探す・メモを取る時に必ず開くアプリ」
- **意思決定の高速化**：ゲーム選択にかかる時間の短縮
- **体験の向上**：「ゲーム体験がより豊かになった」という感想

### 9.3 ユーザー行動の変化
- ゲーム選択時の迷い時間の短縮
- 中断したゲームの復帰率向上
- 新しいゲームの発見・挑戦頻度の増加
